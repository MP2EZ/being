## Using Haptic Features Safely

### üõ°Ô∏è Trauma-Informed Usage

#### Understanding Body Consent
Haptic feedback involves physical contact through your device. We understand this can be concerning for some users, especially those with trauma histories related to unexpected touch.

**Your Rights**:
- **Complete Control**: Enable, disable, or modify haptic feedback anytime
- **No Pressure**: All therapeutic benefits available without haptic feedback
- **Clear Communication**: Always know what haptic feedback will occur and when
- **Immediate Stop**: Instantly disable haptic feedback if needed

#### Managing Trauma Responses
If you experience distress during haptic feedback:

1. **Immediate Actions**:
   - Press and hold the power button to stop all haptic feedback
   - Use voice command: "Stop haptic feedback"  
   - Navigate to quick settings and disable haptics

2. **Ongoing Support**:
   - Contact our trauma-informed support team
   - Adjust settings for gentler or no haptic feedback
   - Use alternative access methods for all features

### üö® Crisis Accessibility

#### Emergency Haptic Override
During mental health crises, haptic feedback can provide crucial accessibility:

- **Emergency Patterns**: Strong, clear haptic feedback for crisis button access
- **Battery Reserve**: Crisis haptics work even in low battery situations  
- **Universal Access**: Crisis haptics function regardless of your normal settings
- **Immediate Recognition**: Distinctive patterns help identify emergency features

#### Crisis Accessibility Features
- **3-Second Rule**: Crisis features accessible within 3 seconds from any screen
- **Bypass Settings**: Emergency features work even if haptic feedback is disabled
- **Clear Confirmation**: Strong haptic confirmation when emergency resources are accessed
- **Screen Reader Priority**: Crisis announcements take priority over other feedback

## Troubleshooting Accessibility Issues

### üîß Common Issues and Solutions

#### Haptic Feedback Not Working
**Possible Causes**:
- Device haptic settings disabled
- Being. haptic consent not given
- Battery saving mode active
- Device not compatible with haptic feedback

**Solutions**:
1. Check device haptic settings (Settings ‚Üí Haptic/Vibration)
2. Review Being. haptic consent (Settings ‚Üí Accessibility ‚Üí Haptic Feedback)
3. Disable battery saving mode temporarily
4. Use visual and audio alternatives instead

#### Screen Reader Conflicts
**If screen reader and haptic feedback interfere**:
1. Adjust screen reader verbosity settings
2. Use Being.'s screen reader optimization mode
3. Customize announcement timing in accessibility settings
4. Contact support for device-specific adjustments

#### Haptic Patterns Too Strong/Weak
**Intensity Adjustment**:
1. Navigate to Settings ‚Üí Accessibility ‚Üí Haptic Feedback
2. Use the intensity slider to adjust
3. Test patterns with the "Try It" button
4. Save your preferred settings

#### Crisis Features Not Accessible
**Emergency Troubleshooting**:
1. Ensure emergency override is enabled (default: ON)
2. Test crisis button accessibility in settings
3. Verify emergency contact information is current
4. Contact crisis support immediately if needed: 988
